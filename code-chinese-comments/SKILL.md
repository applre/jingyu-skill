# 代码中文学习注释

为代码的每一行添加中文注释，帮助初学者理解代码结构和 Python 概念。

## 核心原则

注释的目标是**教学**，不是文档。读者是正在学习编程的人，需要知道每一行"是什么"和"做什么"。

## 注释格式

在每行代码末尾用行内注释标注，格式为 `# [类型标签] 说明文字`。

### 类型标签

新定义出现时，必须在注释开头用方括号标注类型：

| 标签 | 用于 | 示例 |
|------|------|------|
| `[变量：字符串]` | 字符串变量 | `name = "hello"  # [变量：字符串] 用户名` |
| `[变量：整数]` | 整数变量 | `count = 0  # [变量：整数] 计数器` |
| `[变量：布尔]` | 布尔变量 | `is_ready = True  # [变量：布尔] 是否就绪` |
| `[变量：列表]` | 列表 | `items = []  # [变量：列表] 存放结果` |
| `[变量：字典]` | 字典 | `config = {}  # [变量：字典] 配置信息` |
| `[变量：列表[字典]]` | 复合类型 | `tools = [{}]  # [变量：列表[字典]] 工具定义` |
| `[变量：对象 ← 调用 X() 函数]` | 函数返回的对象 | `client = Foo()  # [变量：对象 ← 调用 Foo() 函数] 创建客户端实例` |
| `[变量：类型 ← 调用 X() 函数]` | 函数返回值赋给变量 | `result = get()  # [变量 ← 调用 get() 函数] 获取结果` |
| `[定义 X() 函数]` | 函数定义 | `def foo():  # [定义 foo() 函数] 处理数据` |
| `[调用 X() 函数]` | 调用函数（无返回值） | `load_dotenv()  # [调用 load_dotenv() 函数] 加载环境变量` |
| `[类]` | 类定义 | `class Agent:  # [类] Agent 主类` |
| `[装饰器]` | 装饰器 | `@app.route  # [装饰器] 路由注册` |
| `[常量]` | 全大写常量 | `MAX = 100  # [常量] 最大重试次数` |

### 非定义行

不引入新名字的行不需要类型标签，直接写说明：

```python
if x > 0:                    # 如果 x 大于 0
    return x                 # 返回 x
for item in items:           # 遍历每个元素
    print(item)              # 打印元素
```

### import 行

import 行需要说明三件事：**来源分类**、**模块功能**、**在本文件中的用途**。

来源分类使用以下前缀：
- `标准库` — Python 自带的模块（如 os、sys、json、subprocess）
- `第三方库` — 通过 pip 安装的外部包（如 requests、anthropic、flask）
- `项目内部` — 当前项目中自己写的模块

```python
import os                              # 标准库：操作系统接口，用于读取环境变量、获取工作目录
import subprocess                      # 标准库：子进程管理，用于执行 shell 命令
import json                            # 标准库：JSON 编解码，用于解析 API 响应

from anthropic import Anthropic        # 第三方库：Anthropic 官方 SDK，用于调用 Claude API
from dotenv import load_dotenv         # 第三方库：从 .env 文件加载环境变量到 os.environ
from flask import Flask, request       # 第三方库：Web 框架，Flask 是应用实例，request 获取请求数据

from utils.helpers import format_text  # 项目内部：自定义工具函数，用于格式化文本输出
```

## 注释内容要求

- 用中文解释，技术术语可保留英文（如 API、JSON Schema、token）
- 说明这行代码"做什么"和"为什么"，而不是把代码翻译成中文
- 对初学者可能困惑的语法（如 f-string、列表推导、解包）额外解释
- 注释对齐：同一代码块内的注释尽量右对齐，视觉整洁

## 不要做的事

- 不要删除或修改已有的代码逻辑
- 不要添加代码注释以外的内容（如 docstring、类型注解）
- 不要对空行或纯注释行重复注释
- 不要在注释中使用 emoji
