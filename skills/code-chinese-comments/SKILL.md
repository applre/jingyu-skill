# 代码中文学习注释 v0.1.0

为代码添加**两层注释**，帮助初学者理解代码结构和编程概念：

- **块注释**（Block Comments）：每个逻辑块前的多行框注释，概括"这一块是什么"和"为什么这样设计"
- **行内注释**（Inline Comments）：每行代码末尾的单行注释，标注类型和具体作用

读者可以先通过块注释快速了解整体结构，再按需深入行内注释看细节。

---

## 多语言适配

根据文件扩展名自动适配注释语法和类型术语。

### 注释符号

| 语言 | 行注释 | 块注释框线 |
|------|--------|-----------|
| Python (.py) | `#` | `# ╔═══╗` |
| JavaScript/TypeScript (.js/.ts/.tsx) | `//` | `// ╔═══╗` |
| Go (.go) | `//` | `// ╔═══╗` |
| Rust (.rs) | `//` | `// ╔═══╗` |
| Java/C/C++ (.java/.c/.cpp) | `//` | `// ╔═══╗` |
| Shell (.sh/.bash) | `#` | `# ╔═══╗` |
| HTML (.html) | N/A | `<!-- ╔═══╗ -->` |
| CSS (.css) | N/A | `/* ╔═══╗ */` |

### 类型术语映射

不同语言使用各自的术语，保持对读者自然：

| 概念 | Python | JS/TS | Go | Rust |
|------|--------|-------|----|------|
| 有序集合 | 列表 | 数组 | 切片 | Vec |
| 键值对 | 字典 | 对象 | map | HashMap |
| 文本 | 字符串 | 字符串 | string | String/&str |
| 导入 | import | import/require | import | use |
| 类/结构 | 类 | 类/接口 | 结构体 | 结构体/trait |
| 装饰器 | 装饰器 | 装饰器 | N/A | 属性宏 |

### import/导入行的来源分类

所有语言通用三类前缀：
- `标准库` — 语言自带（Python: os; JS: fs; Go: fmt; Rust: std）
- `第三方库` — 包管理器安装（pip/npm/cargo）
- `项目内部` — 当前项目中自己写的模块

## 第一层：块注释

在每个逻辑块（类、函数组、配置区域）前插入框注释，用 `╔═══╗` 格式包裹。

### 格式（Python 示例，其他语言换注释符）

```python
# ╔══════════════════════════════════════════════════════════════════════════════╗
# ║ 第 N 块：标题                                                               ║
# ║                                                                            ║
# ║ 2-3 句话概括这一块"是什么"和"为什么这样设计"。                                ║
# ║ 如果与前面文件中的某块相同，标注"与 sXX 相同，可以跳过"。                      ║
# ╚══════════════════════════════════════════════════════════════════════════════╝
```

### 规则

- 每个文件通常 4-9 个块，按代码从上到下编号
- 标题简短（如"导入与环境初始化"、"TaskManager 类"、"Agent 循环 + REPL"）
- 正文说明"是什么"和"为什么"，不是翻译代码
- 如果该块与系列中前面文件的某块结构一致，明确标注"与 sXX 相同，可以跳过"，帮助读者跳过重复内容
- 对本文件的核心新增内容，块注释应更详细，解释设计决策和关键机制

### 典型分块方式

| 块 | 内容 |
|------|------|
| 导入与环境初始化 | import、env、client、常量 |
| 核心类/函数 | 本文件的核心新概念 |
| 基础工具函数 | 可复用的工具函数（通常可标注"与 sXX 相同"） |
| 注册/配置 | 工具注册表、路由表、中间件等 |
| 主循环/入口 | main 函数、event loop、REPL |

## 第二层：行内注释

在每行代码末尾用行内注释标注，格式为 `注释符 [类型标签] 说明文字`。

### 类型标签（以 Python 术语为例，其他语言参照术语映射表）

新定义出现时，必须在注释开头用方括号标注类型：

| 标签 | 用于 | 示例 |
|------|------|------|
| `[变量：字符串]` | 字符串变量 | `name = "hello"  # [变量：字符串] 用户名` |
| `[变量：整数]` | 整数变量 | `count = 0  # [变量：整数] 计数器` |
| `[变量：布尔]` | 布尔变量 | `is_ready = True  # [变量：布尔] 是否就绪` |
| `[变量：列表]` | 列表/数组 | `items = []  # [变量：列表] 存放结果` |
| `[变量：字典]` | 字典/对象 | `config = {}  # [变量：字典] 配置信息` |
| `[变量：列表[字典]]` | 复合类型 | `tools = [{}]  # [变量：列表[字典]] 工具定义` |
| `[变量：对象 ← 调用 X() 函数]` | 函数返回的对象 | `client = Foo()  # [变量：对象 ← 调用 Foo() 函数] 创建客户端实例` |
| `[变量 ← 调用 X() 函数]` | 函数返回值 | `result = get()  # [变量 ← 调用 get() 函数] 获取结果` |
| `[定义 X() 函数]` | 函数定义 | `def foo():  # [定义 foo() 函数] 处理数据` |
| `[调用 X() 函数]` | 调用函数（无返回值） | `load_dotenv()  # [调用 load_dotenv() 函数] 加载环境变量` |
| `[类]` | 类/结构体定义 | `class Agent:  # [类] Agent 主类` |
| `[装饰器]` | 装饰器/属性宏 | `@app.route  # [装饰器] 路由注册` |
| `[常量]` | 全大写常量 | `MAX = 100  # [常量] 最大重试次数` |

### 非定义行

不引入新名字的行不需要类型标签，直接写说明：

```python
if x > 0:                    # 如果 x 大于 0
    return x                 # 返回 x
for item in items:           # 遍历每个元素
    print(item)              # 打印元素
```

### import/导入行

说明三件事：**来源分类**、**模块功能**、**在本文件中的用途**。

```python
import os                              # 标准库：操作系统接口，用于读取环境变量、获取工作目录
from anthropic import Anthropic        # 第三方库：Anthropic 官方 SDK，用于调用 Claude API
from utils.helpers import format_text  # 项目内部：自定义工具函数，用于格式化文本输出
```

```javascript
const fs = require('fs');              // 标准库：文件系统，用于读写配置文件
const express = require('express');    // 第三方库：Web 框架，用于创建 HTTP 服务器
const { db } = require('./lib/db');    // 项目内部：数据库连接模块
```

## 注释内容要求

- 用中文解释，技术术语可保留英文（如 API、JSON Schema、token）
- 说明这行代码"做什么"和"为什么"，而不是把代码翻译成中文
- 对初学者可能困惑的语法额外解释
- 注释对齐：同一代码块内的注释尽量右对齐，视觉整洁

## 不要做的事

- 不要删除或修改已有的代码逻辑
- 不要添加代码注释以外的内容（如 docstring、类型注解）
- 不要对空行或纯注释行重复注释
- 不要在注释中使用 emoji
